# Future Scholarly Communication Programme

## Purpose

This programme extends Sourceright from reference and submission screening into a broader agent-first publishing architecture. It documents the target for a future open platform where a research project is represented as a machine-readable, queryable, auditable knowledge object, while familiar human-facing outputs such as manuscripts, preprints, blog posts, presentations, audio, video, and monographs remain generated views over the same record.

The working editorial concept is **Transforming Health**: a cross-disciplinary, open source publishing project focused on improving health and related social outcomes. The programme is intended to support article-level verification, journal-specific submission readiness, reporting checklist assessment, preprint-first workflows, configurable peer review, reproducibility assessment, contribution credit, and knowledge graph evolution over time.

## Strategic premise

The static manuscript should become one presentation layer over a structured research package. The package should include article text, references, data, code, protocols, ethics and governance metadata, reporting checklists, author and contributor roles, peer review, editorial decisions, post-publication updates, dissemination outputs, and provenance.

Human editors, reviewers, authors, readers, and external systems should be able to inspect the same underlying record at different levels of detail. Agents should be able to act on the same record through explicit contracts and audit logs.

## Core questions answered by this programme

- What additional features would make Sourceright and a Transforming Health platform competitive with emerging agentic publishing systems?
- What requirements, designs, contracts, and tracks are needed to move from citation-integrity screening to a full publishing platform?
- Which AI-assisted features are emerging in the literature and current platforms but not yet represented in Sourceright?
- Which open journal, manuscript-management, preprint, overlay, and peer-review platforms should be covered or tracked?
- How should journal-specific article requirements be sourced, contracted, assessed, and optionally implemented?
- How should reporting checklists be discovered, combined, deduplicated, assessed, and optionally implemented?
- How can manuscript, reference, data, and metadata packages become machine-readable by default?
- How should peer review, editorial input, human verification, translation, reproducibility, and dissemination become credited contributions?

## Programme documents

- `requirements.md`: MoSCoW requirements for the platform and Sourceright extension surface.
- `design.md`: architecture and workflow diagrams, including agent-first workflow orchestration.
- `contracts.md`: machine-readable contract families for manuscript packages, journal requirements, checklists, workflows, contribution credit, and knowledge graphs.
- `platform-coverage.md`: open manuscript, journal-management, preprint, overlay, and review platform coverage map.
- `ai-assisted-features.md`: AI-assisted feature gap analysis and recommendations.
- `track-map.md`: granular Conductor tracks created from this programme.

## Claim boundary

This programme is a roadmap and design contract. It does not claim that Sourceright currently performs full manuscript submission, peer review, agentic editorial decision-making, full reproducibility reruns, or public preprint hosting. Any future implementation must keep support claims tied to explicit tests, evidence, and human review gates.
