# GitHub-Side Governance Additions Test Matrix

| Scenario | Acceptance |
| --- | --- |
| Branch protection | Required checks and bypass rules are observed or listed as settings tasks. |
| Copilot | Coding-agent availability is observed or documented as blocked. |
| Renovate | App/config is low-noise and not duplicated by Dependabot PR config. |
| Code scanning | CodeQL, Scorecard, Dependabot alerts, and cargo/npm audits are tracked. |
| Coverage | Codecov/Coveralls/public coverage history decision is recorded. |
| Labels and milestones | Security, dependency, registry, plugin, provider, and external-proof issue routing is observed or listed as a settings task. |
| Review loop | `$conductor-review` runs and local fixes are applied. |
