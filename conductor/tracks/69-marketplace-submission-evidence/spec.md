# Marketplace Submission Evidence Spec

## Goal

Create one evidence model for host marketplaces and client directories so
public claims stay tied to accepted listings, not local intentions.

## Contract

This track is complete only when each relevant host has a recorded state:

- accepted with URL, version, date, install metadata, and submission evidence;
- prepared with repository metadata and blocking requirement; or
- deferred with explicit trigger and claim boundary.

The evidence model must cover Official MCP Registry, Glama, Smithery, VS Code
Marketplace/Open VSX, Microsoft AppSource, LibreOffice extensions, Zotero, OJS,
and GitHub/Copilot surfaces.

## Claim Boundary

Prepared metadata is not public marketplace acceptance. A release note, README,
or docs page must not imply installability until the corresponding evidence row
records an accepted public listing or a validated local package.
