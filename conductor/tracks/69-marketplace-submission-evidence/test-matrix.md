# Marketplace Submission Evidence Test Matrix

| Area | Required check |
| --- | --- |
| Accepted | URL, version, date, install metadata, and external evidence are recorded. |
| Prepared | Repository metadata and blocking requirements are recorded without availability claims. |
| Deferred | Revisit trigger and claim boundary are recorded. |
| Release docs | Release-status and publishing docs use the same evidence state. |
| Policy tests | Public docs cannot claim installability without an accepted listing or validated local package. |
| Review | `$conductor-review` checks marketplace wording before release notes are published. |
