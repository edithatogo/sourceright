# Track 93 - Agent-First Editorial Workflow Engine

## Goal

Define an agent-first workflow engine in which submission, validation, preprint, review, editorial, dissemination, and monitoring workflows are configurable, auditable, and contract-based.

## User outcome

Editors can configure workflows ranging from fully human review to human review with agent support, parallel human-agent comparison, public community review, and agent-only non-decision screening.

## Scope

- Define workflow policy profiles: human-only, agent-assisted, agent-comparative, public community review, automated non-decision screening, and fully automated screening for explicitly non-editorial tasks.
- Define task contracts for intake, requirement sourcing, checklist sourcing, metadata extraction, reviewer matching, technical validation, reproducibility checks, review briefing, agent review, review synthesis, decision-letter drafting, presentation generation, and post-publication monitoring.
- Require audit logs for every agentic action.
- Require human approval for final editorial decisions unless a journal explicitly configures a non-decision task.
- Support parallel workflows for comparison and evaluation.

## Out of scope

- Hidden autonomous editorial decisions.
- Unlabeled agent peer review.
- Sending confidential manuscripts to external AI systems without policy and consent.

## Completion criteria

- Workflow profiles are documented.
- Agent task contracts are documented.
- Audit and human approval boundaries are documented.
- Parallel human-agent comparison is supported at the design level.
