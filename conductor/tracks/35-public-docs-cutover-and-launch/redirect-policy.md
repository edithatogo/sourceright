# Redirect and Canonical-Link Policy

## Canonical URL Policy

The **Astro/Starlight site** deployed at `https://edithatogo.github.io/sourceright/`
is the sole canonical public docs surface. Every page served from this domain is
the authoritative version.

| Concern               | Policy                                                              |
|-----------------------|---------------------------------------------------------------------|
| Canonical surface     | `https://edithatogo.github.io/sourceright/`                         |
| Canonical base path   | `/sourceright/`                                                     |
| Archival source       | `docs/src/` — not served publicly, kept for regression and history  |
| mdBook build          | Retired from public deployment; `book.toml` kept for reference only |

## mdBook URL Redirection

If any external links still point to a former mdBook deployment (e.g., a
`github.io` path without `/sourceright/` or a `/book/` path), the following
strategy applies:

1. **GitHub Pages 404 fallback.** A `404.html` in the Pages root performs
   client-side redirect mapping old mdBook paths (e.g., `/book/guides/foo.html`)
   to the corresponding Astro path (`/sourceright/guides/foo/`).

2. **Path mapping rules.**
   - Strip any `/book/` prefix.
   - Replace `.html` suffix with `/` (Astro clean URLs).
   - Prepend `/sourceright/`.
   - Example: `/book/guides/quickstart.html` → `/sourceright/guides/quickstart/`

3. **SUMMARY.md-based redirect table.** The `docs/src/SUMMARY.md` document
   hierarchy is the authoritative source for mapping old mdBook paths. Every
   entry in SUMMARY.md must have a counterpart in the Astro site sidebar.

## Canonical Link Headers

Every Astro page must include a `<link rel="canonical">` tag pointing to its own
absolute URL on the canonical domain. This is handled by Starlight's built-in
SEO support when `site` is configured in `astro.config.mjs`.

```js
// docs-site/astro.config.mjs — already configured
export default defineConfig({
  site: 'https://edithatogo.github.io/sourceright',
  base: '/sourceright/',
  // ...
});
```

No additional `<link rel="canonical">` injection is required; Starlight
generates it automatically from the `site` + `base` + page slug.

### Verification

- Run `npm run build` in `docs-site/` and inspect `dist/**/index.html` for the
  presence of `<link rel="canonical" href="https://edithatogo.github.io/sourceright/...">`.
- The CI pages workflow already builds the site; canonical link presence can be
  verified with a grep in the deploy workflow.

## SEO Considerations

| Factor                | Handling                                                           |
|-----------------------|--------------------------------------------------------------------|
| Canonical tags        | Auto-generated by Starlight from `site` + `base` config            |
| `robots.txt`          | Starlight generates a permissive `robots.txt`; verify it is present |
| Sitemap               | Starlight generates `sitemap-index.xml` and page-level sitemaps    |
| `meta description`    | Each page frontmatter should include a `description` field         |
| Redirects             | 301-style semantic via canonical tags; client-side 404 fallback    |
| Duplicate content     | Avoided: only one public surface exists; archival source is not indexed |

## Non-Goals

- The mdBook build is **not** maintained as a secondary public surface.
- No server-side redirect infrastructure is required (GitHub Pages is static).
- Browser/bookmark redirects are best-effort via the 404 page approach.
