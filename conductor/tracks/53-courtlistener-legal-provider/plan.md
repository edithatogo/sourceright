# CourtListener Legal Provider Plan

## Current State (as of branch triage)

### What exists
- **Legal citation model**: Full implementation in `src/legal.rs` with `LegalCitationReport`, `LegalCitationRecord`, `LegalCitationType`, `LegalProvider`, `LegalProviderCandidate`, and `LegalCitationIssue` types.
- **CourtListener as enum variant**: `LegalProvider::CourtListener` is defined in the `LegalProvider` enum (line 67), alongside `CaselawAccessProject`, `Austlii`, `LegislationRegister`, and `Other`.
- **Austlii provider candidate**: Only Australian court providers are generated by `provider_candidates_for_court()` — CourtListener is **not yet** emitted as a candidate for US/other courts.
- **Plugin manifest**: `plugins/manifests/legal.courtlistener.toml` exists with status `planned_public_api`, reads/writes `sourceright.legal_citation_report`.
- **No provider fixtures**: No CourtListener-specific fixture files exist in `provider-fixtures/` or `fixtures/`.
- **No CourtListener tests**: No tests exercise CourtListener as a live or fixture-backed provider.

## Implementation Plan

### Slice 1: Mapping and provider candidates
1. Add `provider_candidates_for_court()` entries for US federal courts (SCOTUS, US Circuit, US District).
2. Map `jurisdiction_for_court()` for common US court abbreviations.
3. Emit `LegalProvider::CourtListener` as candidate for US courts with appropriate confidence.

### Slice 2: Fixtures
4. Create `provider-fixtures/courtlistener/` directory.
5. Add fixture files:
   - `success.json` — known US case resolved via CL API.
   - `no-match.json` — citation that yields no CL result.
   - `ambiguous.json` — citation matching multiple CL records.
   - `api-error.json` — simulated rate-limit or auth failure.
6. Add deterministic Rust test cases using fixture data.

### Slice 3: Live smoke
7. Add opt-in live smoke (`COURTLISTENER_API_KEY` env gate).
8. Add rate-limit awareness and cache headers.

### Slice 4: Documentation
9. Update `docs/src/legal-roadmap.md` with CourtListener status.
10. Update plugin manifest status from `planned_public_api` to `fixture_tested` after fixtures are in place.

### Slice 5: Review
11. Run `cargo test` on legal module.
12. Run `$conductor-review`.
13. Apply local fixes; defer any legal advice/compliance claims.
