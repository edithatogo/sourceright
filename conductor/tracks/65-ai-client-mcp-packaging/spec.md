# AI Client MCP Packaging Spec

## Goal

Make Claude, Codex, GitHub Copilot, and generic MCP-client use explicit
packaging targets instead of treating MCP metadata as a catch-all plugin claim.

## Contract

This track is complete only when:

- Claude Desktop, Codex, and generic MCP client install snippets are documented
  against the local stdio contract.
- GitHub Copilot support is described as repository coding-agent preparation
  unless a real Copilot extension or agent package exists.
- Official MCP Registry, Glama, and Smithery states stay separated as accepted,
  prepared, or deferred with versioned evidence.
- Client guides prove read-only discovery and dry-run write behavior without
  adding a hosted HTTP claim.

## Claim Boundary

Do not describe Sourceright as a Claude, Codex, or Copilot plugin until the
target host has an installable package or accepted listing. The current product
claim is MCP-compatible local stdio plus client configuration guidance.
