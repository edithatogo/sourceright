# 08 — Technical-preview launch packaging

Prepare launch-facing text and checklists.

Tasks:
1. Add/update README sections:
   - who it is for;
   - what it does;
   - what it does not claim;
   - benchmark status;
   - demo links;
   - pilot invitation.
2. Add limitations page.
3. Add FAQ.
4. Add pilot feedback issue templates if absent.
5. Add release-note draft for technical preview.
6. Keep positioning accurate:
   - technical preview / pilot-ready;
   - not production institutional product;
   - not externally benchmarked SOTA.
7. Run docs build/tests.

Do not change core behavior unless required by docs accuracy.
