# 03 — Docs and Rust API docs

Improve market-facing docs and docs.rs credibility.

Tasks:
1. Add or expand practical docs:
   - quickstart;
   - worked author preflight;
   - worked editorial triage;
   - artifact/schema guide;
   - live provider configuration;
   - limitations;
   - benchmark interpretation.
2. Add crate-level Rust docs.
3. Add doc comments to high-value public types/functions.
4. Add examples where safe and not too broad.
5. Run:
   - `cargo doc --no-deps`
   - `cargo test`
   - docs build if available.

Keep docs accurate. Do not overclaim production readiness or SOTA.
